Bagunça Organizada

É bem provável que você começou a escrever algo e salvou com um nome legal e depois pensou em algo melhor mas não queria perder a versão anterior.

Talvez salvou assim:

Trabalho.odt
Trabalho final.odt
Trabalho final final.odt

Bem… para algo casual, até funciona bem.

Mas se você precisa fazer isso com frequência, vai acabar com uma grande quantidade de arquivos numa bagunça organizada.

Sistema de Controle de Versão

Uma melhor opção é ter um sistema especializado em gerenciar versões no desenvolvimento de qualquer arquivo.

As vantagens são muitas, começando pelo controle de histórico que permite desfazer facilmente algo que não agradou, podendo recuperar a versão anterior rapidamente. Trabalhar em equipe fica super tranquilo, permitindo até que todos alterem o mesmo arquivo sem criar um caos sem solução.

Algo super legal também é poder ter versões diferentes ao mesmo tempo. E o nível de segurança vai ser o melhor possível.

Git

O Git é uma opção para Sistema de Controle de Versão.

O legal desse SCV é ser distribuído, de código aberto e gratuito, além de ser super fácil de aprender e super rápido, e podemos usar para qualquer tamanho de projeto.

Ser distribuído quer dizer que todos os arquivos estão onde você está, então se você estiver trabalhando em equipe, todos os membros da equipe terão uma copia completa de todos os arquivos.

Ser de código aberto quer dizer que você tem acesso ao código fonte desse sistema e até pode modificar como quiser, além de outras coisas mais.

Gratuito quer dizer gratuito haha.
