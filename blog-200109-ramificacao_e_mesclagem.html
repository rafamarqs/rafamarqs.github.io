Ramificação e mesclagem

20.01.09

Imagine ter um repositório, já tendo algumas confirmações, você cria um novo ramo e faz confirmações nesse novo ramo. Aí você volta ao ramo inicial e cria um novo ramo a partir desse e faz algumas confirmações. Agora temos duas ramificações que precisam se tornar uma novamente.

Até o momento você deve saber como executar essas etapas seguindo as publicações anteriores. Vamos fazer uma mesclagem e para isso precisamos estar no no ramo que irá receber os arquivos para mesclagem.

git checkout master

e o comando de mesclage é simples:

git merge nomeDoRamo

agora faça a confirmação.

Já que não precisamos mais do ramos que foi mesclado, vamos excluir:

git branch -d nomeDoRamo

Agora vamos fazer com o outro ramo, lembrando que agora o ramo principal já tem confirmações que não estão no outro ramo, diferente do primeiro caso.

Faça os mesmos passos feitos antes.

É possível que aconteça de existir modificações no mesmo lugar nas duas ramificações e isso quer dizer que não é só adicionar o novo conteúdo mas que é necessário escolher qual irá ser usado.

Se esse for o caso o git irá mostrar quais arquivos precisam ser vistos e ao abrir esses arquivos verá uma marcação mostrando as versões que existem. É possível até apagar as duas e fazer uma totalmente nova e confirmar.
