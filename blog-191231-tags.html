Tags

19.12.31

Ultima publicação do ano de 2019 com algo bem legal que é identificar algum ponto especifico do repositório com uma tag. É claro que em algum momento você precisa colocar uma numeração do seu código, como a versão v1 e depois v1.1 e etc.

O comando é: git tag

Apenas no formato acima, esse comando mostra todas as tags já criadas.

Tipos

Antes de criarmos uma tag temos que conhecer as duas opções de tags: lightweight ou annotated. Uma tag lightweight é só uma identificação simples em um commit. A tag annotated tem muito mais opções como nome, e-mail, data, mensagem…

Tag lightweight

De forma simples essa tag é uma soma de verificação do commit. Use apenas o comando: git tag v0.1

Veja o resultado dessa tag usando: git tag show v0.1

Tag annotated

Essa tag tem mais informações. Use o comando: git tag -a v1.0 -m ‘mensagem’

Veja o resultado dessa tag usando: git tag show v0.1

Deletar uma tag

Se precisa deletar a tag use a opção -d, o comando fica assim: git tag -d v0.1
