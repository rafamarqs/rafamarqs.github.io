Fazendo alterações no repositório

19.10.12

Você já deve ter um repositório em sua maquina. As atualizações vão acontecer normalmente e vamos precisar registrar essas mudanças quando achar que seja o momento.

Cada arquivo do seu repositório pode estar em dois estados: tracked e untracked. Quando o arquivo está tracked quer dizer que ele já tem uma versão registrada no Git. Quando o arquivo está untracked quer dizer que o Git não conhece esse arquivo.

Assim que você fizer suas edições nos arquivos, o Git identifica como arquivos modificados.

Conferindo o estado dos arquivos

Use o comando git status para ver os estado dos arquivos.

Rastreando novos arquivos

Para começar a registrar novos arquivos digite:

git add nome.txt

Esse comando irá adicionar o arquivo nome.txt ao repositório.

ou:

git add *

Esse ultimo adiciona todos os arquivos ao repositório.

Preparando arquivos modificados

Se você editar um arquivo que está sendo rastreado pelo Git e fizer o comando git status, irá ver que o Git identifica esse arquivo como modificado e não preparado para confirmação (commit).

Para preparar esse arquivo é só fazer o comando git add igual quando fazemos para rastrear novos arquivos. Após esse comando verifique com git status.

O arquivo editado agora está preparado para fazer o commit.

Ignorando arquivos

Vamos ter arquivos que não vai fazer sentido serem rastreados pelo Git. Talvez seja arquivos criados automaticamente por uma IDE ou outra coisa. Nesse caso precisamos criar um arquivo chamado .gitignore.

Crie um arquivo chamado .gitignore e adicione os nomes dos arquivos. Criar esse arquivo antes de iniciar o repositório é uma boa ideia, pode evitar que um commit com arquivos desnecessários seja feito.

Confirmando as edições

Depois de colocar os arquivos na área de preparação, podemos fazer o nosso commit.

git commit -m ‘mensagem de confirmação’

A mensagem de confirmação é um texto descritivo do commit, escolha algo que faça sentido para essa confirmação.

Ignorando a preparação

É possível evitar fazer o git add e ir direto para o git commit, mas isso só funciona para os arquivos que já são rastreados pelo Git. É só adicionar a opção -a.

git commit -a -m ‘mensagem de confirmação’

Removendo arquivos

Se tiver arquivos que não deseja mais, o comando git rm é a solução ideal. Esse comando também remove o arquivo do seu diretório (sua pasta).

git rm nome.txt

O comando acima irá remover o arquivo nome.txt.

Lembre de fazer a confirmação depois.
