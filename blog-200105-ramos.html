Ramos

20.01.05

Se já teve vontade de testar algo mas sem riscos de perder o que já fez, a solução é criar um novo ramo. O novo ramo é totalmente separado do ramo principal.

Sem duvida é um dos recursos mais importantes do git. A forma do git criar um novo ramo é muito leve, sendo bem rápido. Criar novos ramos é recomendável e depois de prontos você pode fundir de volta ao principal, e isso pode ser feito diversas vezes no mesmo dia.

Dominar esse recurso vai mudar toda sua forma de desenvolver seus projetos.

Resumindo

Antes de mais detalhes sobre ramos, vamos voltar ao modo de como o git armazena seus dados.

Quando um commit é feito, o git armazena um objeto de confirmação que tem um ponteiro para os arquivos que você preparou. Esse objeto tem o nome, e-mail, a mensagem e os ponteiros para o commit.

Criando um ramo

O que acontece quando um novo ramo é feito? Isso cria um novo ponteiro para você se movimentar.

Se quiser criar um ramo de teste: git branch teste

Isso cria um novo ponteiro para o mesmo commit em que está.

E como o git sabe qual ponteiro usar? Ele tem um ponteiro extra chamado HEAD que aponta para o ponteiro.

Indo para outro ramo

Se quiser ir para outro ramo é só executar:

git checkout nomeDoRamo
